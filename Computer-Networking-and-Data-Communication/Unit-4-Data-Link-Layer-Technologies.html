1. Data Link Layer (DLL) Functions

PYQ Focus (6 Marks)

Illustrate the functions of the data link layer. Explain the different functions of Data Link Layer.

The Data Link Layer (Layer 2 of the OSI Model) transforms the physical layer—a raw transmission facility—into a reliable link. It is responsible for node-to-node delivery.

A. Core Functions of the Data Link Layer PYQ

To score 6 marks, list and explain these 5 functions:

1. Framing

The DLL divides the stream of raw bits received from the Network Layer into manageable data units called Frames. It adds a header (source and destination addresses) and a trailer (error detection bits) to the data.

2. Physical Addressing

If frames are to be distributed to different systems on the local network, the DLL adds a header defining the sender and receiver's physical MAC Address (Media Access Control address).

3. Flow Control

If the sender's transmission rate is much faster than the receiver's processing rate, the receiver will get overwhelmed and drop data. Flow control mechanisms dictate how much data can be sent before waiting for an acknowledgment.

4. Error Control

The DLL ensures reliability by detecting and retransmitting damaged, lost, or corrupted frames. It uses a trailer called the FCS (Frame Check Sequence) to check for errors.

5. Access Control (MAC)

When two or more devices are connected to the same communication link, MAC protocols determine exactly which device has control over the link at any given time to prevent data collisions.

Easy Word Explanation (The Factory Analogy)

Imagine a factory sending glass bottles. The Physical Layer is the truck driving. The Data Link Layer is the packaging department: they pack bottles into safe boxes (Framing), write the local delivery address (Physical Addressing), make sure the truck isn't overloaded (Flow Control), and check for broken bottles upon arrival (Error Control).

2. Error Detection and Correction

PYQ Focus

Explain Checksum error detection with a suitable diagram/example. Explain CRC method of Error Detection with suitable example. Explain Hamming code correction technique with suitable example.

When data travels across a physical medium, electrical noise can flip a 0 to a 1, causing an error.

  • Single-Bit Error: Only one bit of a given data unit is changed.
  • Burst Error: Two or more bits in the data unit are changed (common in wireless networks).

A. Checksum Error Detection Asked 3 Times

Checksum is a mathematical error detection mechanism based on binary addition.

Sender Side Process

  1. The data is divided into equal-sized segments (e.g., 8-bit or 16-bit blocks).
  2. All segments are added together using 1's complement arithmetic.
  3. The final sum is inverted (1's complement) to create the Checksum.
  4. The sender attaches this Checksum to the data and sends it.

Receiver Side Process

  1. The receiver divides the received message into the same segments.
  2. It adds all segments plus the received Checksum.
  3. If the final inverted result is ALL ZEROS (0000), the data is accepted. If there is even one '1', the data is corrupted and rejected.
6-Mark Example

Checksum Example

Let data be two 4-bit blocks: 1010 and 0101.

SENDER

Addition: 1010 + 0101 = 1111

Checksum (Invert sum): 0000

Send data + checksum:
1010, 0101, 0000

RECEIVER

Receiver adds them:

1010 + 0101 + 0000 = 1111

Receiver inverts sum: 0000

Result is 0, so NO ERROR.

B. Cyclic Redundancy Check (CRC) PYQ

CRC is a highly robust error detection method based on binary polynomial division.

Process:
  1. A predetermined divisor (generator polynomial) is agreed upon by both sender and receiver (e.g., 1011).
  2. The sender appends n-1 zeros to the data (where n is the length of the divisor).
  3. The sender performs binary division (using XOR operation) of the new data by the divisor.
  4. The remainder obtained is the CRC. The sender replaces the appended zeros with the CRC and sends the frame.
  5. The receiver performs the same division. If the remainder is 0, there is no error.

Easy Word Explanation (The Password)

The sender divides the data by a secret number. The remainder is attached like a password. The receiver divides the whole packet by the same secret number. If the math divides perfectly without a remainder, the data is uncorrupted.

C. Hamming Code Correction Technique PYQ

Error detection only tells you there is an error. Error correction tells you exactly which bit flipped so the receiver can fix it automatically without asking for retransmission.

The Concept: Hamming code adds multiple redundant bits (Parity bits: P1, P2, P4, etc.) to specific positions in the data block.

Formula to find redundant bits (r): 2r ≥ m + r + 1 (where m = number of data bits)
Process Example (for 4-bit data 1011):
  • We need m=4 bits. Using the formula, we find we need r=3 parity bits (since 23 ≥ 4 + 3 + 1, i.e., 8 ≥ 8). Total bits = 7.
  • Parity bits are placed at positions that are powers of 2 (Positions 1, 2, and 4). Data bits go in positions 3, 5, 6, 7.
  • The parity bits are calculated based on even or odd parity of specific combinations of bits.
  • At the receiver, parity equations are checked again. If an error occurred, the results of the equations point to the exact decimal position of the wrong bit! The receiver simply flips that single bit to correct it.

3. Multiplexing: FDM, TDM, and CDMA

PYQ Focus (6 Marks)

Analyse the need for multiplexing. Compare TDMA and FDMA. Write their advantages.

A. Need for Multiplexing PYQ

In data communication, laying individual wires for every single device is incredibly expensive. Often, the bandwidth of a transmission medium (like an Optical Fiber) is much greater than the bandwidth required by a single device.

Multiplexing is the technique of combining multiple signals into one single composite signal, which is then transmitted over a single shared physical link.

Reduces cabling cost Maximizes high-capacity channels

B. Comparison: FDMA vs TDMA PYQ

Feature FDMA (Frequency Division Multiple Access) TDMA (Time Division Multiple Access)
Basic Concept The total available bandwidth is divided into non-overlapping frequency bands. Each user is assigned a specific frequency. The total bandwidth is shared, but divided into time slots. Each user gets the full bandwidth for a brief time fraction.
Usage of Link All users share the link simultaneously, but use different frequencies. Users take turns using the link, one after the other.
Interference Prone to "crosstalk" (adjacent channels mixing). Requires "Guard Bands" to separate them. Prone to synchronization issues. Requires "Guard Times" between slots.
Example / Advantage FM Radio, Cable TV.
Advantage: Simple to implement, good for analog continuous signals.		 2G GSM Mobile Networks.
Advantage: Highly efficient for digital data; immune to frequency crosstalk.
Easy Word A 4-lane highway where each car gets its own dedicated lane to drive continuously. A single-lane bridge where Cars A, B, and C take turns crossing it for 10 seconds each.

(Note: CDMA - Code Division Multiple Access allows all users to transmit at the same time and on the same frequency by assigning a unique mathematical code to each user).

4. Switching Techniques

PYQ Focus (6 Marks)

Discuss package (packet) switching and mention features. Compare packet switching and circuit switching and write the disadvantage of each.

A. Need for Switching

If we have 100 computers, connecting them directly with point-to-point cables would require thousands of wires (Mesh Topology). Instead, we connect them to a central "Switch" which dynamically routes data. This is switching.

B. Circuit Switching

Definition: A dedicated, physical communication path is established between the sender and receiver before data transfer begins. The path remains reserved for the entire duration of the communication.

Example: Traditional landline telephone networks.

Disadvantage PYQ Highly inefficient for computer data. If the two computers pause transmitting for 5 seconds, the line remains blocked, and bandwidth is wasted because nobody else can use that physical wire.

C. Packet Switching

Definition: The data message is broken down into smaller chunks called Packets. Each packet is given a header containing the destination IP address. Packets travel independently through the network, taking different routes, and are reassembled at the destination.

Features:
  1. No dedicated path is required.
  2. Highly efficient because unused bandwidth is dynamically shared among many users.
  3. Robust: If one router goes down, packets automatically take a different route.

Example: The Internet.

Disadvantage PYQ Packets may arrive out of order or get lost/delayed due to network congestion, making it less ideal for high-quality, real-time voice calls without special protocols.

D. Comparison Table: Circuit vs Packet Switching PYQ

Data TransmissionData is sent as a continuous stream.Data is chopped into independent packets.
Bandwidth UsagePoor efficiency. Bandwidth is reserved even if idle.High efficiency. Bandwidth is shared dynamically.

5. Network Layer & Internet Protocol (IP)

PYQ Focus

Describe the features of the network layer and network layer protocols. Compare IPv4 and IPv6 addressing schemes and write the need for IPv6. Internetworking purpose.

The Network Layer (Layer 3 of OSI) is responsible for the source-to-destination delivery of a packet across multiple networks (links).

A. Features & Functions PYQ

  • Logical Addressing

    The MAC address from DLL only works locally. Network layer adds an IP Address to uniquely identify devices globally across the internet.

  • Routing

    Determining the best, shortest, or most optimal path for a packet to travel from the source to the destination router.

  • Fragmentation

    Breaking down packets into smaller fragments if the underlying Data Link Layer cannot handle large packet sizes (MTU limits).

B. Network Layer Protocols PYQ

  • IP
    IPv4 / IPv6

    The core protocol responsible for addressing and routing packets.

  • ICMP
    Internet Control Message Protocol

    Used by routers to send error messages and operational info (e.g., the ping command uses ICMP).

  • ARP
    Address Resolution Protocol

    Used to map a logical IP address to a physical MAC address.

The Need for IPv6 (PYQ Focus):

The primary reason IPv6 was created is IPv4 Address Exhaustion. With the explosion of smartphones, smart TVs, and IoT devices, the 4.3 billion IPv4 addresses ran out. IPv6 provides enough addresses for every atom on the surface of the earth, ensuring the Internet can keep expanding.

D. Internetworking & Subnetting PYQ

Define Internetworking: It is the practice of connecting multiple distinct computer networks (like LANs, WANs, and MANs) together using hardware like Routers and Gateways, allowing them to communicate seamlessly.

Purpose of Internetworking: To break geographical boundaries. It allows a computer in a local LAN in India to pull down a webpage hosted on a LAN in New York. The ultimate internetwork is the Internet.

6. Routing and Congestion Control

PYQ Focus

Define Routing and types. Define dynamic routing and 2 advantages. Define Congestion control and state its effects.

A. Routing and its Types

Routing is the process of selecting a path for traffic in a network across multiple networks. A Router uses a "Routing Table" to decide where to forward the packet next.

  • Static Routing The network administrator manually types the routes into the router. It is highly secure but impossible to maintain for large networks.
  • Dynamic Routing PYQ The routers use intelligent algorithms (like OSPF or BGP) to automatically discover the network and update their routing tables dynamically by talking to other routers. Advantages:
    1. Automatic Failover: If a cable breaks, router automatically calculates new path.
    2. Scalability: Very easy to expand network; new routers detected automatically.

B. Congestion Control

Definition: Congestion occurs when the number of packets being transmitted through a network is greater than the capacity of the network to handle them (Traffic > Bandwidth). Congestion control is a set of techniques used to monitor and regulate the traffic entering the network to prevent a collapse.

Effects of Congestion PYQ
  • Packet Delay The queue in the router's memory fills up, causing massive delays.
  • Packet Loss Once router's memory buffer is 100% full, it must drop new arriving packets.
  • Deadlock & Retransmissions Dropped packets cause senders to re-transmit, pouring even more traffic into the jammed network, causing total collapse.
Easy Explanation (Traffic Jam)

It is exactly like a massive traffic jam on a bridge. Too many cars (packets) enter. Cars slow down (Delay). Eventually, cars have to be pushed off the bridge (Packet Loss).
