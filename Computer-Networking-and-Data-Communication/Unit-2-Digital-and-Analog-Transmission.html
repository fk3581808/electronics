1. Transmission Modes

PYQ Focus (6 Marks)

Explain the transmission mode in brief. Describe in brief the full duplex transmission mode with a suitable diagram.

A. What is Transmission Mode?

Transmission Mode (also known as Communication Mode) defines the direction of signal flow between two connected devices. It dictates how data is transmitted over the network.

Easy Word Explanation

Transmission mode is simply the "traffic rules" for the data highway. It decides whether cars (data) can go one way, both ways one-at-a-time, or both ways simultaneously.

B. Types of Transmission Modes

1. Simplex Mode

Definition: Communication is strictly unidirectional (one-way). One device can only transmit, and the other can only receive.

Electronics Example: A keyboard connected to a computer, or a traditional TV broadcast.

Advantages: Uses the entire bandwidth of the channel.

2. Half-Duplex Mode

Definition: Communication is bidirectional, but not at the same time. Both devices can transmit and receive, but when one is sending, the other must only receive.

Electronics Example: Walkie-Talkies (you must say "Over" to let the other person know it's their turn to speak).

Highly Important for Exams

3. Full-Duplex Mode

Definition: Communication is bidirectional and occurs simultaneously. Both devices can transmit and receive data at the exact same time. The link contains two separate physical or logical channels—one for sending and one for receiving.

Electronics Example: A standard telephone network or a smartphone call. You can talk and listen at the same time.

6-Mark Exam Diagram Description: (To draw in exam)

Draw two computers (Device A and Device B). Draw two separate arrow lines between them. Top arrow points from A to B (labeled "Direction of data A to B"). Bottom arrow points from B to A (labeled "Direction of data B to A"). Write "Occurs simultaneously" below it.

Advantages: Maximum efficiency and zero delay since devices don't wait for their turn.

2. Digital Transmission & Conversion

PYQ Focus

Explain Digital transmission. Write different ways of digital transmission. Explain Line coding and Block coding. Explain characteristics of any three line coding schemes.

A. What is Digital Transmission?

Digital Transmission is the transmittal of digital signals between two or more points in a communication system. The data can be either digital (like computer files) or analog (like voice), but it must be converted into a digital signal (0s and 1s) before being sent over the channel.

B. Digital-to-Digital Conversion

This is the process of representing digital data (the 1s and 0s generated by the computer) into digital signals (voltage pulses) that can be sent over a wire. It involves two main techniques: Line Coding and Block Coding.

1. Line Coding PYQ Focus

Definition: Line coding is the process of converting digital data (a sequence of bits) into a digital signal (a sequence of voltage pulses).

Easy Word Explanation

It is the language translation between the CPU and the wire. The CPU understands "1 and 0". The wire only understands "High Voltage (+5V) and Low Voltage (0V)". Line coding translates the 1s and 0s into +5V and 0V.

Characteristics of a Good Line Coding Scheme (PYQ):
  • Signal Element vs Data Element: A good scheme minimizes the number of signal changes per bit to save bandwidth.
  • Self-Synchronization: The receiver's clock must be perfectly synchronized with the sender's clock. Good line coding includes "timing" pulses built into the signal.
  • No DC Component: Continuous 1s or 0s create a flat DC signal, which cannot pass through certain electronic components like transformers. Good coding avoids this.
Types of Line Coding:
A. Unipolar Encoding

Uses only one non-zero voltage level.

Ex: Unipolar NRZ. Logic 1 = +V, Logic 0 = 0V. Very simple but suffers from DC component issues.

B. Polar Encoding

Uses two non-zero voltage levels (positive and negative).

Ex: Polar NRZ, RZ, and Manchester Coding. Logic 1 = +V, Logic 0 = -V.

C. Bipolar Encoding

Uses three voltage levels: Positive, Negative, and Zero.

Ex: AMI. Logic 0 = 0V. Logic 1 alternates between +V and -V. Prevents DC components.

2. Block Coding PYQ Focus

Definition: Block coding changes a block of m bits into a block of n bits (where n > m) before line coding is applied. It is also called mB/nB encoding.

Why is it needed? To improve the performance of line coding. It adds extra, redundant bits to the data stream to ensure synchronization and provide error detection.

Example: 4B/5B encoding. It takes 4 bits of user data and converts it into a 5-bit code. The extra bit ensures there are no long strings of continuous zeros, which would cause the receiver to lose clock synchronization.

3. Analog-to-Digital Conversion (ADC)

PYQ Focus

Define Sampling and Quantization. Describe the process of quantization. Explain the Analog to digital converter with suitable diagram and parameters. Explain the flash type ADC with conversion time.

When we want to send an analog signal (like human voice or temperature) over a digital computer network, we must convert it into a digital signal. This is done using an Analog-to-Digital Converter (ADC). The conversion happens in three strict steps (collectively called Pulse Code Modulation or PCM).

A. The 3 Steps of ADC

PYQ

1. Sampling

Definition: Sampling is the process of measuring the instantaneous amplitude (voltage) of the continuous analog signal at discrete, regular time intervals.

Easy Ex: Imagine recording a video of a moving car. A video is just taking 30 still photographs (samples) every second. Sampling is taking "voltage photographs" of the wave very quickly.
Nyquist Sampling Theorem:
To accurately reproduce the signal, the sampling rate (fs) must be at least twice the highest frequency (fmax) of the analog signal.
Formula: fs ≥ 2 fmax
PYQ

2. Quantization

Definition: Quantization is the process of mapping the continuous sampled voltage values to a set of discrete, predefined voltage levels.

The Process:
  1. The total voltage range is divided into a fixed number of zones (levels).
  2. When a sample is taken, its exact voltage (e.g., 3.14V) might not match a predefined level.
  3. The system rounds off or approximates the value to the nearest predefined level (e.g., rounds 3.14V down to 3.0V).
  4. This rounding off introduces a tiny error known as Quantization Error.

3. Encoding

Definition: The process of converting the quantized discrete voltage levels into binary codes (0s and 1s). For example, if level 3 was chosen, it is encoded as 011.

B. Flash Type Analog to Digital Converter PYQ

The Flash ADC is the fastest type of ADC available.

Working Principle:

It uses a series of high-speed comparators that operate simultaneously (in parallel). The analog input voltage is fed to all comparators at once. A resistor ladder divides the reference voltage into different levels. The comparators compare the input with these levels and output a digital code instantly via a priority encoder.

Conversion Time:

Because all comparisons happen in parallel simultaneously, the conversion time is incredibly fast, typically in the range of 1 to 20 nanoseconds (ns).

Drawback:

Requires a huge number of comparators. For an n-bit ADC, it needs 2n - 1 comparators. (An 8-bit Flash ADC needs 255 comparators, making it expensive and bulky).

Diagram Note for Exam: Draw a voltage divider resistor ladder on the left, connected to the negative terminals of multiple Op-Amp comparators. Connect the analog input to all positive terminals. The outputs go into a Priority Encoder block.

C. Parameters of an ADC PYQ

  • Resolution: The smallest change in the analog input voltage that can cause a change in the digital output code.
  • Conversion Time: The total time taken by the ADC to completely convert one analog sample into its equivalent digital code.
  • Accuracy: How close the actual digital output is to the theoretical expected output.

4. Digital-to-Analog Conversion (DAC)

PYQ Focus

Explain the parameters of a DAC. Calculate the resolution of 6-bit and 12-bit DAC using a 10V reference.

Digital-to-Analog Conversion (DAC) is the reverse process. It takes digital data (0s and 1s from the computer) and converts it back into a continuous analog voltage or current. (Example: Your phone's audio jack uses a DAC to convert MP3 files into electrical waves for your earphones).

A. Parameters of a DAC PYQ

  • Resolution: The number of different analog output voltage levels the DAC can produce. It is determined by the number of input bits (n). Total levels = 2n. The percentage resolution is the step size relative to the full scale.
  • Settling Time: The time required for the analog output voltage to stabilize and settle within a specified error band after a digital input code changes.
  • Linearity: Ideally, the relationship between the digital input and analog output should be a perfectly straight line. Deviation from this line is the linearity error.

B. PYQ Numerical Solution: DAC Resolution

Important Numerical

Question:

Calculate the resolution of i) 6-bit DAC and ii) 12-bit DAC in terms of percentage using a 10V reference.

Formulas Needed:

  • Number of steps = 2n - 1 (where n is the number of bits)
  • Step Size (Voltage Res) = Vref / (2n - 1) or Vref / 2n (Using 2n is standard for step size)
  • Percentage Resolution = [1 / (2n - 1)] × 100%
i) For a 6-bit DAC:

Number of bits (n) = 6

Total steps = 26 - 1 = 64 - 1 = 63

Percentage Resolution = (1 / 63) × 100 = 1.587 %

(Optional Voltage Resolution) = 10V / 64 = 0.156 V per step.

ii) For a 12-bit DAC:

Number of bits (n) = 12

Total steps = 212 - 1 = 4096 - 1 = 4095

Percentage Resolution = (1 / 4095) × 100 = 0.0244 %

(Optional Voltage Resolution) = 10V / 4096 = 2.44 mV per step.

(Note: The 12-bit DAC has a much smaller percentage, meaning it is highly precise and accurate compared to the 6-bit DAC).

5. Analog-to-Analog Conversion

Sometimes an analog signal (like voice) needs to be sent over an analog medium (like air) over long distances. Because low-frequency voice signals cannot travel far, we use Modulation (Analog-to-Analog conversion). The low-frequency data is mixed with a high-frequency "Carrier Wave".

Amplitude Modulation (AM)

The amplitude (height) of the high-frequency carrier wave is varied in accordance with the analog data signal.

Frequency Modulation (FM)

The frequency of the carrier wave is varied in accordance with the analog data signal. (Highly resistant to noise, used in FM radio).

Phase Modulation (PM)

The phase of the carrier wave is varied in accordance with the data signal.

6. Important PYQ Numericals

Bandwidth & Sampling problems that frequently appear in exams.

Numerical 1

Minimum Sampling Rate

Question:

A complex low-pass signal has a bandwidth of 200 kHz; calculate the minimum sampling rate for this signal.

Solution:

According to the Nyquist Sampling Theorem, the minimum sampling rate (fs) must be twice the highest frequency (fmax).

For a low-pass signal, the bandwidth is equal to its maximum frequency.

fmax = 200 kHz

Minimum Sampling Rate (fs) = 2 × fmax

fs = 2 × 200 kHz = 400 kHz (or 400,000 samples per second)

Numerical 2

Baud Rate and Levels

Question:

Calculate the number of levels, baud rate, and bandwidth for sending 3 bits at 3 Mbps with a carrier frequency of 10 MHz.

Solution:

Number of bits sent per signal element (r) = 3 bits/baud.

Data Rate / Bit Rate (N) = 3 Mbps = 3,000,000 bps.

1. Number of Levels (L):

L = 2r = 23 = 8 levels

(This means it's using an 8-level modulation scheme like 8-PSK or 8-QAM).

2. Baud Rate (S):

Baud rate is the number of signal elements sent per second. Formula: S = N / r.

S = 3 Mbps / 3 = 1 MBaud

(1 Mega Baud or 1,000,000 signals/second).

3. Minimum Bandwidth (B):

For basic theoretical ASK/PSK transmission, Bandwidth is roughly equal to the Baud Rate.

B ≈ S = 1 MHz

(The 10 MHz carrier frequency is just the frequency the signal sits on, it doesn't change the bandwidth calculation).
